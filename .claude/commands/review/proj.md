# 프로젝트 전체 구조 리뷰

프로젝트 아키텍처, 폴더 구조, 핵심 파일을 분석합니다.

## 리뷰 프로세스

### 1단계: 프로젝트 구조 파악
```bash
tree -L 3 -I node_modules
find . -name "*.json" -type f | head -20
ls -la src/
```

**확인 사항:**
- 디렉토리 구조와 네이밍 규칙
- 패키지 설정 (package.json, tsconfig.json, vite.config.ts 등)
- 진입점 파일 (main.ts, index.ts 등)

### 2단계: 핵심 파일 샘플링
- src/main.ts, src/index.ts 등 진입점
- 각 모듈/폴더의 대표 파일
- 설정 파일 (config, env 등)

### 3단계: 아키텍처 평가

**구조적 평가 항목:**
| 항목 | 배점 | 체크포인트 |
|------|------|-----------|
| **정확성** (25점) | - | 타입 안전, 모듈 간 타입 일치도 |
| **가독성** (20점) | - | 폴더 구조 직관성, 네이밍 일관성 |
| **안정성** (20점) | - | 모듈 경계 명확성, 순환 의존도 |
| **성능** (15점) | - | 번들 크기, 로드 순서, 캐싱 |
| **보안** (10점) | - | 의존성 관리, 권한 분리, 환경변수 |
| **유지보수성** (10점) | - | 모듈화, 단일 책임, 테스트 기초 |
