# 개별 파일 코드 리뷰

특정 파일의 코드를 상세하게 리뷰합니다.

**사용법:**
```
/review:file src/main.ts
/review:file website/js/main.js
/review:file package.json
```

## 리뷰 프로세스

### 1단계: 파일 기본 정보 수집
- 파일 경로 및 크기
- 파일 타입 (TypeScript, JavaScript, JSON, CSS, HTML 등)
- 라인 수
- 의존성 분석 (import/require)

### 2단계: 코드 분석

**검사 항목:**
| 항목 | 배점 | 체크포인트 |
|------|------|-----------|
| **정확성** (25점) | - | 논리 오류, 타입 오류, 엣지 케이스 |
| **가독성** (20점) | - | 변수명, 함수명, 함수 크기, 주석 |
| **안정성** (20점) | - | 에러 처리, null 체크, try-catch |
| **성능** (15점) | - | 불필요 연산, 반복 최적화, 알고리즘 |
| **보안** (10점) | - | 입력 검증, 민감 정보, OWASP |
| **유지보수성** (10점) | - | DRY, 테스트 가능성, 함수 책임 |

### 3단계: 상세 평가

**함수/메서드별:**
- 복잡도 분석
- 입출력 검증
- 에러 처리

**클래스/모듈별:**
- 책임 분리
- 상태 관리
- 인터페이스 설계

## 출력 형식

```
## 📄 파일 코드 리뷰: src/main.ts

### 📊 파일 정보
- 경로: src/main.ts
- 크기: X KB
- 라인 수: Y 줄
- 타입: TypeScript

### ⭐ 총점: SS / 100

| 항목 | 점수 | 평가 |
|------|------|------|
| 정확성 | X/25 | ✅/⚠️/❌ |
| 가독성 | X/20 | ✅/⚠️/❌ |
| 안정성 | X/20 | ✅/⚠️/❌ |
| 성능 | X/15 | ✅/⚠️/❌ |
| 보안 | X/10 | ✅/⚠️/❌ |
| 유지보수성 | X/10 | ✅/⚠️/❌ |

### 잘된 점 👍
- 함수가 단일 책임을 잘 따름
- 에러 처리가 명확함
- ...

### 개선 필요 ⚠️
- 함수 길이가 길어서 분할 추천
- null 체크 누락
- ...

### 심각한 문제 ❌
- 순환 의존성 발견
- 미처리 Promise
- ...

### 🔍 상세 분석

#### 함수 분석
- **functionName()**: 복잡도 높음, 테스트 필요
- ...

#### 클래스 분석
- **ClassName**: 책임이 많음, 분할 권고
- ...

#### 의존성
- 순환 의존도: 없음/있음
- 외부 의존성: X개
```
