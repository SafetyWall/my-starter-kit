# git diff 기반 코드 리뷰

현재 변경사항(staged/unstaged)을 리뷰합니다.

<commit_analysis>

## 변경 파일 수집

```bash
git diff HEAD --name-only
git diff HEAD --stat
git diff HEAD
git status --short
```

**git diff** 출력에서:
- 변경된 파일 목록
- 각 파일의 라인별 변경사항 (+ 추가, - 삭제)
- 컨텍스트 정보

</commit_analysis>

## 리뷰 프로세스

### 1단계: 변경 파일 파악
- git status / git diff로 변경 범위 파악
- 파일 개수, 라인 수 통계 수집

### 2단계: 변경 내용 분석
각 변경 파일에 대해:
- **수정 라인**: 추가/삭제된 코드 검토
- **컨텍스트**: 주변 코드와의 관계 파악
- **의도**: 커밋 메시지와 맞는지 확인

### 3단계: 리뷰 항목별 채점
| 항목 | 배점 | 체크포인트 |
|------|------|-----------|
| **정확성** (25점) | - | 버그, 로직 오류, 엣지 케이스, 타입 불일치 |
| **가독성** (20점) | - | 네이밍, 구조, 주석 필요성 |
| **안정성** (20점) | - | 에러 처리, null/undefined, 예외 케이스 |
| **성능** (15점) | - | 불필요 연산, 반복 최적화, 메모리 누수 |
| **보안** (10점) | - | OWASP, 민감 정보, 입력 검증 |
| **유지보수성** (10점) | - | 중복 제거, 단일 책임, 테스트 가능성 |
